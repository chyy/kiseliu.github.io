Spark2.0.0 PySpark依赖包py4j升级问题

Posted on 2016-08-01   |   In Spark   |  

上周把Spark更新到2.0.0的时候，运行pyspark接口，总是有问题，报错信息如下：
```python
\Traceback (most recent call last):
File “/Users/lyj/Programs/kiseliugit/MyPysparkCodes/test/spark2.0.py”, line 5, in \
spark = SparkSession.builder.master(“local”).appName(‘test 2.0’).config(conf=SparkConf()).getOrCreate()
File “/Users/lyj/Programs/Apache/Spark2/python/pyspark/conf.py”, line 104, in init
SparkContext._ensure_initialized()
File “/Users/lyj/Programs/Apache/Spark2/python/pyspark/context.py”, line 243, in _ensure_initialized
SparkContext._gateway = gateway or launch_gateway()
File “/Users/lyj/Programs/Apache/Spark2/python/pyspark/java_gateway.py”, line 116, in launch_gateway
java_import(gateway.jvm, “org.apache.spark.SparkConf”)
File “/Library/Python/2.7/site-packages/py4j/java_gateway.py”, line 90, in java_import
return_value = get_return_value(answer, gateway_client, None, None)
File “/Library/Python/2.7/site-packages/py4j/protocol.py”, line 306, in get_return_value
value = OUTPUT_CONVERTERtype(answer2:, gateway_client)
KeyError: u’y’
```
`因为用的是PyCharm，之前配置PySpark环境的时候就折腾了一番，所以出了这个问题，我就觉得难不成是环境配置的问题。然后又因为Spark2.0.0整合了SparkContext和SQLContext成为新的入口SparkSession，所以在网上查了很多关于SparkSession的例子，结果又是搞了很久也没解决这个问题。

继在好几个技术群里求助未果之后，我只好到Stack Overflow上求助，过了两天终于有人回复我了，附上链接https://stackoverflow.com/questions/38637988/how-could-i-write-the-right-entry-point-in-spark-2-0-program-actually-pyspark-2 。万能的Stack Overflow，必须赞一个👍，虽然我总是问一些小白问题，但是都有人回答。

根据答友提供的解决方案，只需要更新一下Python的py4j库，pip install py4j –upgrade，然后问题就解决了。说到底还是对pyspark底层不了解，即使我知道Spark2.0.0的根目录下的./python/lib/py4j-0.10.1-src.zip 从0.9更新到0.10.1了，错误提示也说到了py4j这个库，我也还是没想到只需要升级一下py4j就没问题了。
