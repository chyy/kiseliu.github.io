情感分析

Posted on 2016-09-08   |   In Natural Language Processing   |  

情感分析：根据文本所表达的含义和情感信息将文本分为褒扬或贬义的两种或多种类型。

情感分析是特殊的文本分类问题，既有一般文本分类的共性问题，也有其特殊性。例如情感信息表达的隐蔽性，多义性和极性不明显等。领域相关性强。数据不平衡较为明显。
批评等负面评价通常比较隐晦，商品评论几乎都是正面评价，只有很少量数据是差评，数据不平衡会影响有监督分类器的效果。有些评论有星级，但是星级和实际内容不相符，会误导我们的分类器。很多评论(影评)有很大段话，即说好，也有说不好的地方，也会误导分类器，很难评判是好是坏。甚至还有一些文字和感情无关，只用几句话总结态度，对分类带来困扰。因此，比起通常的主题聚类，情感分类要难很多。按照论文中的说法，电影影评是最难分类的，通常只有65.83%。

读论文(Thumbs up? Sentiment Classification using Machine Learning Techniques)：
论文主要讨论了情感分析很难，我们要用哪种分类器，哪种特征选取的方法，哪种权重计算方式来做呢？

数据：IMDB的数据，电影评论数据是天生的学习集，不仅有评论，还有评分。而且电影评论是最难的任务，如果把这个做好了，其他的就容易多了。
分成正，负，中三种态度。在文章中，只取正负，并且选取评论，每个作者最多只选择20篇。语料库有752负，1301正，144个评论者。

如何选取特征：
肉眼选取：人为选定一些正面，负面的词。准确率只有58%，64%，很不好。
统计筛选：正面评论和负面评论中出现的比较多的词，69%。

结论：
学习集：随机抽取正面的700个样本，700个负面样本。
(1) 在整个语料库中出现至少4次的词；unigram，频数；
(2) unigram，是否出现；比1好
(3) unigram+bigram(至少出现7次)，是否出现；
(4) bigram(至少出现7次)，是否出现；
(5) unigram+POS
=>unigram比bigram和二者混合都更好

Cut-based classification

先从原始的分类信息中抽取出来一些和情感分类有关的信息，然后尝试对这些信息进行分类。
极性数据集：各1000个样本，312个作者，每个作者不超过20篇；
默认的分类器：朴素贝叶斯和SVM
相关性训练集：具有强烈情感的5000条单句训练相关性侦测器。
相关性侦测器：给句子打分，如何构造？用上述5000个句子训练缺省的极性分类器。
句子之间的打分如何产生：
如果一个句子表达了极性的情绪，和他相邻的句子也可能表达相应的情绪，离得越近，相关性越强。

可以开始计算了：
生成cut-based 图，然后计算每条边的权值。通过cut-based classification得到一组分类情况，然后保留该种情况的边，之后进行排序，取得分最高的若干边。然后我们就把评论浓缩为这几个句子，作为训练数据训练缺省的极性分类器。
