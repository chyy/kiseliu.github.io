机器翻译

Posted on 2016-11-21   |   In Natural Language Processing   |  

机器翻译源起：
第一代——词汇转换为主的方法
词汇有一词多义问题；考虑上下文，出现第二代。还有词汇的顺序问题；
第二代——句法分析为主的方法
但是也不好，不能用有限的规则全部概括。
第三代——语义分析为主的方法
觉得句法分析还不能解决语义的问题，考虑对语义进行分析，引入到机器翻译中。

随着统计方法的兴起：使用语料库，
平行语料：是句子对齐，而不是词对齐(EM算法)。

噪声信道与统计机器翻译
原语言 -> 噪声信道 -> 目标语言，这里原语言和目标语言和我们要翻译的原语言和目标语言相反。

翻译三原则:信(准确)，达(充分)，雅(优美)

1 词对齐问题
2 可能性太多，性能差

1 词汇翻译概率分布：最大似然估计
使用EM算法
2 对齐函数
满足函数条件

IBM翻译模型1
$$ p(e,a|f)=\frac { \E }{ { ({ l }_{ f }+1) }^{ { l }_{ e } } } \prod _{ j=1 }^{ { l }_{ e } }{ t({ e }_{ j }|{ f }_{ a(j) }) } $$
