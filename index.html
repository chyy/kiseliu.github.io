Spark2.0 MLlib 更新说明

Posted on 2016-07-28   |   In Spark

Machine Learning Library (MLlib) Guide

MLlib 是Spark的机器学习库，它旨在让机器学习算法具有可扩展性，并且很容易实现。它提供的工具如下：

  • ML Algorithms: 常用的机器学习算法，如分类，回归，聚类，和协同过滤
  • Featurization: 特征抽取，特征转换，特征降维，和特征选择
  • Pipelines: 构建，评价，和调优的ML Pipelines工具
  • Persistence: 保存和加载算法，模型和Pipelines
  • Utilities: linear algebra, statistics, data handling, etc.

Announcement: DataFrame-based API is primary API

基于RDD的MLlib API现在处于维护模式。
对于Spark 2.0，spark.mllib库中基于RDD的APIs已经进入了维护模式。现在 Spark主要的机器学习API是spark.ml库中基于DataFrame的API。

这意味着什么？
  • MLlib依然支持对spark.mllib库中基于RDD的API进行bug维护。
  • MLlib不会对基于RDD的API增加新的特性。
  • In the Spark 2.x releases, MLlib will add features to the DataFrames-based API to reach feature parity with the RDD-based API. 在Spark 2.x的发行版本中，MLlib将会对基于DataFrames的API增加特性来达到和基于RDD的API进行功能校验。
  • 在达到功能校验之后(粗略估计是Spark2.2)，基于RDD的API会被废除。
  • 基于RDD的API可能会被移动到Spark3.0中。
为什么MLlib要转而提供基于DataFrame的API？
  • 比起 RDDs，DataFrames 提供了更多对用户友好的API。DataFrames 的许多好处包括 Spark 数据源，SQL/DataFrame查询，Tungsten 和 Catalyst 优化，跨语言的一致性API。
  • MLlib中基于DataFrame的API在机器学习算法和跨语言方面提供了一致的API。
  • DataFrame使得使用ML pipelines更加便捷，尤其是特征转换。

Github多账号以及Pages搭建问题

Posted on 2016-06-23   |   In Github

今天终于搭建好了自己的第一个博客，眼看工作马上就满一年，可是自己却从未写过什么。身边的同学，同事都用CSDN，但是我每次查资料进到CSDN的网页，看到比较糟糕的文字和公式排版，就根本看不进去内容。直到前两天看到别人分享的有关机器学习的文章，页面的清爽让我感到眼前一亮。然后我注意到域名是github.io，就通过搜索查到了github pages，然后开始萌生在github上开始建立自己的博客的想法。

想法总是美好的，可是实际操作起来不搞个1-2天是根本解决不了问题的，尤其是在我的github 2个帐号冲突的问题没有解决，以及我从来没有在本地仓库和远程仓库进行过任何交互的情况下，还想要结合Jekyll来搭建自己的github pages。

废话不多说，直接进入正题。

Github pages搭建问题

我的搭建过程主要是参考 痞子达 同学的这篇博客，博客写的很详细，但是我第一次按照这个流程走的时候，还是遇到了问题。

  • username.git.com VS username.git.io
    username.git.com 这个域名已经被废除了，作者的文章是在 username.git.io 启用之前写的，所以这里指明一下。我当时排查错误的时候，怀疑过是不是应该把username.git.io 改成 username.git.com (原谅我这种奇葩的想法)。

  • master VS gh-pages
    直到我今天搞了半天，终于定位到是分支的问题，我记得之前按照上述博客中的步骤提交的时候，报错信息里提到了master 和 gh-pages。今天我从头到尾又试了一次，也出现了同样的问题(我当时的操作是git push origin master，可是我的.git配置的是gh-pages分支 ，自然是要出现问题的)。

我们在git clone https://github.com/username/username.github.io.git 的时候，一定要确保 clone 的分支是master，如果clone下来的是 gh-pages，最后程序是不能提交到master分支，只能提交到gh-pages分支，但是gh-pages分支的内容不会在https://username.git.io 网页上显示出来(根据jekyll官网上说是因为我们建立的是个人主页，如果是项目主页，站点分支会被放到gh-pages分支上，不过我表示不知道怎么建立项目主页)。

具体修改的方法是在username.git.io目录下，选择Settings —> Branches，然后将默认分支改为master，点击update即可。

GitHub 多个帐号冲突的问题

前天花了很长时间解决帐号冲突的问题，之前对Github不是很了解(其实现在也不是很了解)，只是注册了帐号，然后在本地安装了git，没有真正地使用过git。在这里推荐 廖雪峰 老师的git教程，我个人觉得说的还是比较清楚的，新手不妨看看。

解决该问题的主要思想就是先生成两组密钥，建立两个账号的安全连接；然后提交代码的时候，指定不同的帐户采用不同的密钥，具体可以参考如下链接(没想到百度经验还是有点用的)：同一台电脑设置多个公钥与不同Github帐号交互。

如果在提交代码的时候，遇到错误说，该次提交被某个用户拒绝了，就说明你远程仓库的地址不是本次提交的用户的，你可以通过 git remote set-url origin github.com:githubname/repository.git 这个命令把远程仓库origin的地址修改为本次提交的用户的地址。(远程仓库的名称是可以修改的，根据你自己的进行设置)
